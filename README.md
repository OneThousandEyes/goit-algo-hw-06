### Висновки:

- BFS (Breadth-First Search) обходить граф рівнями, використовуючи чергу.
У неваговому графі він гарантовано знаходить шлях з мінімальною кількістю ребер між початковою та цільовою вершинами.
- DFS (Depth-First Search) обходить граф вглиб, використовуючи стек.
Він повертає перший знайдений шлях, який залежить від порядку обходу сусідів і не обов’язково є найкоротшим.
- Для даного мережевого графа BFS і DFS можуть повертати різні шляхи, що добре видно як за: 
  - довжиною шляху, 
  - так і за порядком відвідування вершин (visit order).
- Visit order наочно демонструє різницю в логіці алгоритмів:
  - BFS розширюється «хвилею» від стартової вершини,
  - DFS спочатку досліджує одну гілку максимально глибоко.
- Отже, BFS доцільно використовувати, коли важлива мінімальна кількість переходів,
а DFS — для повного або глибинного дослідження структури графа.
